🔴 P-03  Magic Strings — Inconsistencia en TipoMovimiento
El enum TipoMovimientoEnum existe y es usado correctamente en InventarioService. Sin embargo, VentasService (L165, L233) y ComprasService (L176, L237) crean registros de HistorialMovimientoInventario con cadenas literales:
❌ VentasService.cs L165
TipoMovimiento = "SALIDA",
// ComprasService.cs L176
TipoMovimiento = "ENTRADA",
✅ Debería ser
TipoMovimiento = TipoMovimientoEnum.SALIDA.ToString(),
// Usando el enum centralizado
Esto crea riesgo: si el enum se renombra, la BD tendrá valores mixtos ("Salida" vs "SALIDA") sin error de compilación.
🔴 P-02  Sin Transacciones — Race Condition en Ventas y Compras
En CrearVentaAsync: se valida stock → se crea la venta → se descuenta inventario en tres operaciones lógicas separadas. Si el proceso cae entre la validación de stock y el descuento, el inventario quedará inconsistente. Mismo caso en RecibirCompraAsync donde se actualiza stock + historial + estado de compra.
❌ Sin transacción
_context.Ventas.Add(venta);
// actualiza stocks...
await _context.SaveChangesAsync();
✅ Con transacción
using var tx = await _context.Database
.BeginTransactionAsync();
// operaciones...
await _context.SaveChangesAsync();
await tx.CommitAsync();
🟡 P-05  HandleResult duplicado — Solución: ControllerBase genérico
Los 4 controladores tienen exactamente el mismo bloque de ~10 líneas copiado. La solución correcta es un ApiControllerBase : ControllerBase que todos los controllers hereden:

if (result.Success) return Ok(result.Data);

}

public abstract class ApiControllerBase
